A long time ago, every day, I would come home from work and think about making the perfect programming language.
It would be an extremely simple language with as few features as possible. 
This language would be the simplest possible language, with as few features as possible:

- Functions and structs only
- Basic control-flow - if/switch/for constructs, with golang like semantics
- Range operator like a..<b
- Non-capturing Lambdas borrowing typescript syntax like const x = () => {} 
- Unions stolen from typesript, like type X = Foo | Bar | Baz | number | null
- Ability for a union value's type to be narrowed solely via if-checks (also stolen by TypeScript)
- All data is a value type unless explicitly indirected I.e pointers

It was not a very well thought out language.
The Odin programming language has 95% of the things that I would want in this language.
As such, I have stopped thinking about that other programming language since.
